But what do I mean that the array is now visible to the function?

Let‚Äôs back up one step. Remember when I talked about the scope of a function? All the variables (i.e. named types - <type> <name> = <value>; - ) that are defined inside the body (scope) of a function are all the memory that is visible (available) to that function.

To be a little clearer, in our example, we have defined a variable (array) of type char (my_text) in the main function. This array is visible to main only. If we want to do something with my_text‚Äôs value in (let‚Äôs say) our function my_print(), such as printing it to the console, we have to pass it as an argument to that function, utilizing call by value (or call by reference, but this is a bit more advanced concept ‚Äì we‚Äôll talk about this in the next lesson) i.e. passing the value of our array to an array that is visible to my_print() ‚Äì none other than my_arg as is defined in our example*.
